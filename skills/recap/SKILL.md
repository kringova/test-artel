# recap

Собрать читаемые итоги работы за период — что сделано и зачем, сгруппированно по проектам. Для 360-ревью, отчётов руководителю и саморефлексии («что я вообще успел» — в потоке это теряется). По умолчанию **read-only**: ничего в vault не меняет. Самодостаточно: можно дать любому агенту.

## Когда сам

Когда пользователь прощается или уходит («пошла спать», «на сегодня всё», «до завтра») — собери и дай **итоги дня** без отдельной просьбы.

## Период

- «итоги дня» → сегодня
- «неделя» → последние 7 дней
- «месяц» / «квартал» / «год» → соответствующий календарный/скользящий период
- произвольный диапазон по датам («с 1 по 15 мая»)

## Что собрать

1. **Закрытые задачи** (`status: done`), у которых дата закрытия попадает в период. Дата закрытия = `updated` задачи (или дата git-коммита, который выставил `done`, если git доступен — точнее). Сгруппировать по проектам.
2. **Записи логов сессий** («## Что сделано в сессии <дата>» в карточках проектов) за период — там детали, контекст, формулировки.
3. (опц.) Созданные задачи и то, что сейчас в `doing` — для полноты, отдельным коротким блоком.

## Как подать (формат итогов)

Не дамп задач, а **результат**:

- **Заголовок** — период + одна строка «итого: N задач закрыто по M проектам».
- **Хайлайты** (2–3 сверху) — самое значимое за период; для 360 это то, что показывают руководителю. Брать по импакту/RICE и по тому, что меняет продукт или метрику.
- **По проектам** — 2–5 буллетов на проект: «что достигнуто и зачем» (outcome, а не «закоммитил X»). Источник — `summary` задач + логи сессий, формулировать по-человечески.
- **Тон** — фактический, в терминах результата и влияния, без воды и приукрашивания.

## 360-ревью (по запросу)

Если просят «для ревью» / «по компетенциям» — добавить раздел, где результаты сгруппированы по компетенциям, которые они демонстрируют: delivery (довёл до прода), ownership (взял и закрыл без напоминаний), качество (тесты/верификация/фиксы), влияние на продукт/метрику, инициатива. Каждую компетенцию — подкрепить 1–2 конкретными результатами периода.

## Опции

- **Сохранить** — если просят «сохрани итоги», записать в `recaps/<период>.md` (например `recaps/2026-W24.md`, `recaps/2026-06.md`). Это единственный случай, когда navык пишет в vault.

## Правила

- Read-only по умолчанию — задачи и карточки не менять (кроме явного «сохрани»).
- Честно: только реально закрытое (`done`) или явно в работе; не приукрашивать и не выдумывать.
- Язык — русский.

## Самопроверка

- [ ] Собраны done-задачи за период (по дате закрытия) + записи логов сессий
- [ ] Итог структурирован: хайлайты → по проектам, в терминах результата
- [ ] Ничего в vault не изменено (если не просили «сохрани»)
- [ ] Только реальные результаты, без приукрашивания

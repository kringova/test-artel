# sync

Загрузить актуальное состояние всех проектов и задач из vault и выдать сжатую сводку. Навык **read-only** — ничего не изменяет. Самодостаточен: можно дать любому агенту.

## Где vault

Корень текущего репозитория — папка с `CLAUDE.md` и `projects/`. Если рабочая директория глубже, поднимись до корня.

## Что прочитать

1. `_rules.md` — конвенции vault (как ведутся проекты и задачи).
2. `_lessons.md` — журнал инцидентов (храповик ошибок): какие ошибки были и какой механизм/правило теперь их ловит. Читать обязательно — это «обучение на ошибках» между сессиями.
3. `_requests.md` — открытые запросы пользователя (незакрытые `- [ ]`): что обещано и не доделано. Держать в фокусе, закрывать.
4. `_dashboard.md` — общая картина.
3. Для каждой папки в `projects/`:
   - `projects/<slug>/<slug>.md` — статус, приоритет, аудитория, метрики, лог сессий, топ-задача.
   - `projects/<slug>/roadmap.md` — фазы и текущий этап.
   - `projects/<slug>/tasks/*.md` — статусы и RICE. Хватает frontmatter; для скорости можно `grep` по `status:`, `rice_*`, `summary:`, `next_up:`, не читая тела.

RICE считается: `(rice_reach × rice_impact × rice_confidence/100) / rice_effort`.

## Как показать результат

Сжатая сводка на русском. Структура:

```
## 📌 Запланировано — взять сейчас (open + next_up: true), по убыванию RICE
- <тикет> · <проект> · <задача> (RICE N)   ← только если такие есть; иначе блок не выводить

## Проекты
- **<название>** — статус, приоритет, аудитория
  - Топ-задача: <название> (RICE N)
  - Фаза роадмапа: <текущая>

## Активные задачи (todo + doing), по убыванию RICE
| Проект | Задача | Статус | RICE |
|--------|--------|--------|------|

## Недавно завершённые (последние 5)
- <проект> · <задача>
```

Не пересказывай файлы целиком — только ключевые поля. Активные задачи сортируй по убыванию RICE. `done`/`cancelled` в активный список не попадают.

## Запланированные задачи (📌 next_up) — брать без напоминания

Открытые задачи с `next_up: true` — это **очередь «взять в работу»**, помеченная пользователем кнопкой «Запланировать». Выводи их **первым блоком** сводки «📌 Запланировано — взять сейчас» (по убыванию RICE), выделенно.

**После сводки возьми верхнюю в работу сам, без отдельной просьбы** (это действие уже вне read-only sync): пользователь пометил их именно для этого. Учитывай `blocked` и пререквизиты — заблокированную или с незакрытым пререквизитом не бери, назови блокер. Перед стартом переведи задачу в `doing` (гейт входа в работу). Если запланированного нет — работай по обычному приоритету (блок «С чего начать» ниже).

## Когда «В работе» пусто

Если ни одной задачи в статусе `doing` нет — не оставляй пользователя без фокуса, предложи **3 задачи, с которых начать**:

- **Проектов с открытыми (`todo`) задачами меньше 3** → топ-3 открытых задач по убыванию RICE (из любых проектов).
- **Проектов 3 и больше** → по одной задаче из трёх *разных* проектов: у каждого проекта возьми его самую дорогую открытую задачу (макс. RICE), затем выбери три проекта, чьи топ-задачи дороже всех. Фокус распределяется по проектам, а не сваливается в один.

Показывай их блоком «С чего начать» с RICE — выбирает пользователь.

## Правила

- Сам навык — read-only (только сводка). Но если есть запланированные (`next_up: true`) открытые задачи, **после** сводки возьми верхнюю по RICE в работу без напоминания (перевод в `doing`) — см. «Запланированные задачи».
- Ничего не изменять — только читать.
- После сводки (и блока «С чего начать», если `doing` пуст) спросить пользователя, над чем работаем.

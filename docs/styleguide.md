# Стайлгайд Артели

Как писать документацию, чтобы она помогала адресату, а не существовала ради
объёма. Канон Артели; проект может иметь тонкий оверрайд (термины, голос бренда),
не дублируя этот документ.

## 1. Сначала APC, потом текст

Каждый документ начинается с явной шапки — кто адресат и зачем он здесь:

```markdown
<!-- Audience: контрибьютор-агент
     Purpose: понять структуру проекта за 2 минуты, чтобы начать менять код -->
```

Purpose удобно формулировать как **JTBD**: «когда я …, хочу …, чтобы …». Может
быть списком работ. Нет ясных Audience и Purpose — документ не пишется.

## 2. Не смешивать типы (Diátaxis как prior art)

Из [Diátaxis](https://diataxis.fr/) берём один принцип: разные потребности — разные
документы. Не лепи туториал, справку и объяснение в один файл. Туториал ведёт за
руку; how-to решает задачу; reference — сухая правда для справки; объяснение
отвечает «почему». Рамку строим свою (см. карту в [doc-canon.md](doc-canon.md)),
расширяя на агента-читателя.

## 3. Против ИИ-воды

Документ = решение/действие адресата, не текст ради объёма. Запрещено:

- преамбулы «в этой статье мы рассмотрим», «давайте разберёмся»;
- маркетинговый пух: «мощный», «бесшовный», «легко и просто», «революционный»;
- повторы и пересказ заголовка в первом абзаце;
- объяснения очевидного и вода ради длины;
- хедж-фразы «возможно, стоит, как правило» там, где нужен факт.

Правило проверки: если абзац можно удалить без потери для Purpose адресата — удали.

## 4. Лаконичность = DoD-формат

Как DoD задачи описывает наблюдаемый результат, так документ описывает, **что
адресат сможет сделать**, в минимуме слов. Скорее таблица/список, чем простыня.
Скорее пример, чем абзац описания.

## 5. Внутренняя дока — под машинное чтение

Документация для агента (`AGENTS.md`, архитектура) пишется структурно и
машиночитаемо: открытый `AGENTS.md`-формат, явные команды, пути, конвенции. Цель —
агент въезжает без догадок (это снижает требуемый tier модели, см. раздел 4.1
методологии). Никакого нарратива там, где работает список.

## 6. Голос

По умолчанию: настоящее время, активный залог, второе лицо («запусти», а не «можно
запустить»). Термины — единообразно (глоссарий проекта при необходимости). Пиши
для самого занятого и самого нетерпеливого представителя Audience.

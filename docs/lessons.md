# Журнал инцидентов (learning ratchet)

**Audience:** агент и владелец проекта. **Purpose:** превратить «обучение на ошибках» из обещания агента в свойство системы.

Агент (LLM) не учится между сессиями — веса не меняются, контекст сбрасывается. Переживают только **файлы** (правила, память, скиллы) и **механические гейты** (хуки, валидаторы). И прозовые правила доказанно нарушаются: правило можно написать и всё равно нарушить. Значит надёжное «обучение» — не ещё одно напоминание, а **храповик**.

## Принцип храповика

> Повторная ошибка ⇒ **механический гейт**, не напоминание.

- Первый раз — допустимо правило/запись в журнал.
- Повтор класса ошибки — **обязан** механизм, работающий без участия агента: pre-commit-хук, валидатор, структурное изменение флоу, гейт в инструменте.
- Механизм поставлен, а ошибка класса повторилась → чинить **механизм** (он недостаточен), а не обещать «постараюсь».

Журнал читается в начале каждой сессии (навык `sync`) — это и есть точка, где система «вспоминает» уроки вместо агента.

## Формат записи

```
## <дата> — <краткое название инцидента>
- **Что:** что пошло не так (наблюдаемо).
- **Почему:** корневая причина.
- **Механизм:** какой хук/валидатор/структурное изменение теперь это ловит.
          ИЛИ **Дыра:** механизма нет; план, чем закрыть. Признать, что это слабее гейта.
- **Класс:** обобщённый тип ошибки — чтобы распознать повтор.
```

## Пример

```
## 2026-06-13 — Закрытие задач без подтверждения пользователя
- Что: агент сам перевёл задачи в `done`, минуя явный апрув.
- Почему: правило «done — только апрув» жило прозой; на инерции «работа сделана»
  переход в done ощущается естественным.
- Механизм: потолок агента `review`; `done` пишет только человеческий апрув;
  pre-commit-хук блокирует агентский коммит с переходом в `done`.
- Класс: агент присваивает себе финальное действие, зарезервированное за человеком.
```

Инстанс ведёт свой журнал (в llmush — `_lessons.md` в корне vault) с реальными инцидентами; этот файл — формат и принцип для всех, кто ставит фреймворк.

# Архитектура Артели

Артель — не приложение, а **шаблон-репозиторий**: markdown-vault + переносимые
навыки + методология. Кода для сборки нет; «работает» = агент читает файлы и
действует по ним. Источник правды — сами markdown-файлы.

## Структура репозитория

| Путь | Роль |
|---|---|
| `AGENTS.md` | вендор-нейтральный контракт для агента (гейты, навыки, правила) |
| `CLAUDE.md` | шим `@AGENTS.md` для Claude Code |
| `_rules.md` | полная методология ведения vault |
| `_dashboard.md` | Dataview-сводка: проекты, беклог по RICE, канбан |
| `_inbox/` | буфер сырых мыслей «на разбор», по файлу на запись |
| `_templates/` | шаблоны: `task.md`, `project.md`, `docs/` (канон документации проекта) |
| `projects/<slug>/` | карточка проекта, `roadmap.md`, `tasks/` |
| `skills/` | переносимые навыки (markdown-инструкции, пути относительные) |
| `docs/` | документация Артели (методология, роли, гейты, канон, стайлгайд) |
| `dashboard/` | опциональная веб-витрина vault + захват инбокса (`POST /api/inbox`) + апрув задач review→done (`POST /api/approve`) + страницы задач/проектов и поиск |

## Навыки (`skills/`)

`sync` (контекст проектов в сессию) · `backlog` (завести задачу: RICE, роли, грейд) ·
`close` (закрыть задачу + лог + changelog) · `recap` (итоги за период) ·
`inbox` (разобрать `_inbox/`). Самодостаточны: каждый можно дать любому агенту.

## Документация (`docs/`)

Методология (`methodology.md`) — ядро; над ней: `roles.md` (команда),
`workflow.md` (жизненный цикл и гейты), `doc-canon.md` (канон документации, карта
APC), `styleguide.md` (как писать). Справка: `reference.md`, `skills.md`,
`installation.md`, `faq.md`. Полная навигация — в [README](../README.md).

## Как это переносится

llmush — первый пользователь: его `.claude/skills/` — копия `skills/` отсюда,
глобальные `~/.claude/skills/` — варианты с абсолютным путём. Изменения методологии
текут: **artelush → llmush → глобальные**. Канон нейтральный, адаптеры под вендора
(Claude subagents, грейды моделей) — опциональны и тонкие.

Та же независимость — на уровне конфига агента: `AGENTS.md` в корне каждого репо как
канонический контракт, `CLAUDE.md` = `@AGENTS.md`-шим, вендорские папки (`.claude/` и
пр.) — read-only зеркала в `.gitignore`. Правила и причина отложить компилятор — в
[doc-canon.md](doc-canon.md) → «Вендор-агностичность конфига».
